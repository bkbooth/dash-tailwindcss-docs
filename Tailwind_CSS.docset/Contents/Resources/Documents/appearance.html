Customizing

Responsive and State Variants

By default, only responsive variants are generated for appearance utilities.

You can control which variants are generated for the appearance utilities by modifying the appearance property in the modules section of your Tailwind config file.

For example, this config will also generate hover and focus variants:

{
// ...
modules: {
    // ...
-   appearance: ['responsive'],
+   appearance: ['responsive', 'hover', 'focus', 'active', 'group-hover'],
}
}

Disabling

If you don't plan to use the appearance utilities in your project, you can disable them entirely by setting the appearance property to false in the modules section of your config file:

{
// ...
modules: {
    // ...
-   appearance: ['responsive'],
+   appearance: false,
}
}
